# Documentation

Hub files and spoke directories for the skills-engineering-skill project.

## Hubs

Each hub is a concise summary that delegates detail to its spoke directory:

- `ARCHITECTURE.md` → `architecture/`
- `UBIQUITOUS-LANGUAGE.md` → `ubiquitous-language/`
- `TECH-STACK.md` → `tech-stack/`
- `DEVELOPER-WORKFLOWS.md` → `developer-workflows/`
- `USER-EXPERIENCE.md` → `user-experience/`

## Standalone directories

- `adr/` — Architecture Decision Records (numbered files, no hub)
- `plans/` — Implementation plans and specs (no hub)
- `musings/` — Pre-artifact thought capture (no hub)
- `scratchpads/` — Working notes (no hub)
- `troves/` — Research collections (no hub)

Read the hub first, then drill into spokes when you need detail.
